# Issue Tracker

## System

Use GitHub Issues through the `gh` CLI.

## Issue Contract

Issues should include:

- problem
- acceptance criteria
- constraints
- links
- priority
- status

Keep implementation scratch notes out of issues. Put current resume context in `docs/handoffs/`.

## Workflow

Use one issue, one branch, one worktree, and one Codex thread. Start from the issue scope, run the documented checks, update the handoff, and commit or stash before leaving the task.
